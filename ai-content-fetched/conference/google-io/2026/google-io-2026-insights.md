## 一、洞察总结

### 战略层

**Agentic Era 全面到来**：Google 全面转向 Agent-First，从"辅助工具"升级为"任务执行者"。

**全栈 AI 协同**：Infra(TPU) → Models → Platform → Protocols → Products 全链路整合。

---

### 模型层

**Gemini 3.5 矩阵**：
1. Flash：高性价比，即日上线
2. Pro：复杂推理，6月内测
3. Spark：个人 Agent，夏季灰度

**Gemma 4**：2B/4B/26B/31B，Apache 2.0，累计下载 1 亿+。

**Omni**：融合 VEO/Nano Banana/Genie，统一多模态理解与生成。

---

### 平台层

**Antigravity 2.0**：Agent-First IDE，子 Agent 并行，12h 构建 OS。

**Firebase Agent-Native**：SQL Connect 升级，原生 Agent 工具调用。

**AI Studio**：Prompt → App 全流程，零门槛开发。

---

### 协议层

1. **MCP**：工具与数据接入
2. **A2A**：跨团队 Agent 协作
3. **UCP**：标准化商务交易
4. **WebMCP**：浏览器标准，Agent 可操控网页

---

### 应用层

- **Search**：25年最大升级，AI 原生搜索框
- **Chrome**：WebMCP、HTML-in-Canvas、内置 AI API
- **Android**：Agent 模式、Compose 1.10/1.11
- **Flutter**：Dart Cloud Functions、代理式热重载
- **Robotics**：Gemini Robotics 1.6、Physical AI

### TPU Gen 8 详解

**产品定位**
自研 AI 芯片，双芯片方案，训练/推理分离优化。

**核心架构**
```
                    ┌─────────────────┐
                    │   JAX + Pathways │
                    │  (统一训练框架)   │
                    └────────┬────────┘
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
   ┌──────┴──────┐   ┌──────┴──────┐
   │   TPU 80     │   │   TPU 8i     │
   │  (Training)  │   │  (Inference)  │
   │               │   │               │
   │ • 3x 算力提升│   │ • 1500 tok/s │
   │ • 跨集群扩展 │   │ • 2x 能效   │
   └──────────────┘   └──────────────┘
```

**核心能力**

| 能力 | TPU 80 | TPU 8i |
|------|--------|--------|
| 定位 | 大规模训练 | 推理 |
| 算力 | 3x 上代 | 低延迟 |
| 扩展 | 100万+ TPU 集群 | 单卡高效 |
| 能效 | - | 2x 提升 |

**软件栈**
- **MaxText**：JAX 大模型参考实现，数千芯片扩展
- **Tunix**：轻量后训练框架
- **vLLM**：Paged Attention + Prefix Caching + Continuous Batching
- **Torch TPU**：PyTorch 代码无需修改跑在 TPU 上

**关键数据**
| 指标 | 数据 |
|------|------|
| TPU 80 算力提升 | 3x |
| TPU 8i 推理速度 | 1500 token/s |
| 能效提升 | 2x |
| 集群规模 | 100万+ TPU |

## L7: Physical AI 层详解

### VLA 模型架构

**核心突破**
- Vision-Language-Action：视觉语言模型 + 动作模态
- 物理标记替代数字标记
- 理解来自数字 AI 突破（如恐龙玩具识别）

**两种训练路径**
```
┌─────────────────────────────────────────────────────────────┐
│                   机器人训练路径                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐  │
│  │              可仿真任务                              │  │
│  │  仿真器 + 强化学习 → 策略迁移                      │  │
│  │  适合：行走、跑步、跳舞、全身控制                  │  │
│  └─────────────────────────────────────────────────────┘  │
│                            │                               │
│  ┌─────────────────────────────────────────────────────┐  │
│  │              灵巧操作（难以仿真）                    │  │
│  │  远程操作 + 真实世界数据                            │  │
│  │  VR 头显 + 物理标记收集                            │  │
│  └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**关键挑战**

| 挑战 | 状态 |
|------|------|
| 平衡 | 已解决（仿真+RL） |
| 灵巧操作 | 最大难题，未完全解决 |
| 触觉感知 | 硬件不成熟，主要靠视觉 |
| 泛化能力 | 死板，换瓶子就不行 |

**时间预期**
- 5-10 年开始看到更多场景
- 灵巧操作 + 安全是主要障碍
