## 📝 内容总结

**《设计和构建桌面级 Android 体验》演讲总结**

本次演讲由 Google Android 团队的产品经理 Josh Hale、设计师 Jason 和技术专家 Francesco 共同分享，介绍了如何为桌面级 Android 设备（折叠屏、平板、Chromebook 及桌面环境）打造出色的应用体验。

**核心理念：** 桌面级应用不仅仅是手机应用的简单放大，而需充分利用大屏空间，提供真正适配桌面环境的功能与体验。

**设计与体验要点：**
- 基于**窗口尺寸**而非设备屏幕进行设计，利用 Android 的窗口尺寸类断点适配布局
- 采用**多列布局、渐进式信息披露**和**窗格（panes）**结构，扁平化导航、提升信息密度
- 优化字体大小和内容宽度，适配桌面观看距离
- 导航栏随窗口增大从底部迁移至侧边甚至展开

**输入交互优化：**
- 完善**鼠标和触控板**支持，包括悬停状态、I 型光标、右键上下文菜单
- 全面支持**实体键盘**，包括 Tab/方向键导航、Enter/Space 触发、明确焦点样式及标准快捷键

**工具与开发支持：**
- 即将推出面向 AOSP 的**桌面开发者指南**，涵盖键盘鼠标、拖放、多窗口等最佳实践
- **Android Studio** 新增**桌面模拟器**，可测试自由窗口、外接显示屏等场景
- **Jetpack Compose 1.11** 改进了触控板支持，新增手势识别和测试 API

总体而言，团队提供了从设计指南、开发文档到模拟器测试的完整工具链，鼓励开发者从设计初期就以桌面体验为核心，打造真正"桌面级"的 Android 应用。

>> 大家好，我叫 Josh Hale，是 Google 的产品经理，负责 Android 相关工作。Android 是一个跨多种设备的强大操作系统。现在，Android 应用可以运行在各种不同形态的设备上，从折叠屏、平板，到 Chromebook，再到完整的桌面环境。这不仅仅是屏幕变大了，而是桌面级的强大性能。通过为这些设备形态进行优化，你不仅仅是把应用放大，而是在解锁用户创造、工作和娱乐的全新方式。在这些大屏设备上的用户，希望看到真正能利用额外空间的应用。他们期待的不仅仅是把手机应用简单拉伸。应用应该能自然支持键盘和鼠标，流畅处理窗口缩放，并帮助用户提高效率或更深入地沉浸其中。满足用户对功能和体验的期待，就是我们所说的"桌面级应用"。过去构建这类体验需要额外投入。所以，我们一直努力提供更多内置功能，给你所需的工具和指导。当然，一个优秀的桌面级应用不仅仅是代码，设计也同样重要。因此，我们也整理了针对桌面端的新设计指南。这些指南包含大量实用技巧和示例，能帮你思考如何最佳利用空间、创建直观的导航，以及设计适合指针操作的交互方式，比如悬停效果和右键菜单。

其次，我们即将推出面向基于 AOSP 技术开发的桌面开发者指南。可以把它看作是为大屏设备构建顶级应用的"操作手册"。我们会介绍键盘鼠标支持、拖放操作、多窗口处理等最佳实践，以及所有那些让应用在桌面环境中真正精致、得心应手的细节。

我们知道在所有这些不同设备上进行测试很困难。这就是为什么我们要推出一款全新的桌面模拟器。现在你就可以在 Android Studio 中下载使用。这款模拟器提供完整的桌面测试环境，你可以查看应用在自由窗口中的表现，确保键盘和鼠标支持稳定，甚至可以测试连接外接显示屏的场景。这将大大方便你构建和迭代桌面级应用，无需为每种情况都准备物理设备。

我们真的非常期待看到你们开发出的精彩应用。现在我把话筒交给 Jason，让他详细介绍一下设计指南。

>> 谢谢 Josh。在当今的 Android 开发时代，先为移动端设计、再去适配其他设备形态，已经不再是最佳策略。在设计阶段就充分考虑桌面环境，才能确保你的应用能够提供完整的桌面级体验。请记住，用户体验不仅意味着在缩放 UI 时不丢失功能和视觉完整性，还包括用户如何通过触摸、鼠标和键盘与应用进行交互。

下面我们聊聊一些实现最佳桌面体验的基础技巧。

首先，不要针对设备屏幕进行设计，而应该考虑窗口尺寸。窗口是应用实际渲染的屏幕区域，包括全屏模式、所有设备上的分屏模式，以及平板和连接显示屏上的自由窗口模式。Android 提供了一组我们称为"窗口尺寸类"的推荐断点。我们在开发者和设计指南中都会使用这些分类来帮助你调整布局。更多信息请查看屏幕上的链接。

当窗口尺寸增大时，首先寻找机会将内容分段为多列布局，这样用户无需滚动就能看到更多内容。除了重新排列内容之外，还要考虑以下细节调整：

- 使用渐进式信息披露，展示在小窗口中可能被隐藏在点击之后的内容。
- 桌面用户期望更密集的 UI，以持续方式显示更多内容和工具。
- 适当放大字体，以补偿桌面环境较远的观看距离。Android 会尝试设置一个合适的默认显示尺寸，但你可以在设计中进行个性化调整，以获得更好的编辑效果。
- 设置最大内容宽度，避免内容在整个屏幕上被扭曲或过度拉伸。
- 短文本控制在 60 个字符左右，长文本则可以更长。

大屏空间还意味着，你可以使用我们称为"窗格（panes）"的布局容器来扁平化应用结构。就像现实世界中窗户由一块块玻璃组成一样，窗格也可以组成应用窗口的主体区域。窗格可以让应用导航更高效：你可以将窗格并排放置，展示可能覆盖在主视图上方的支撑内容，或者在小窗口中需要深入导航才能看到的内容。

说到导航，如果在小窗口中使用了底部导航栏，当窗口尺寸达到中等及以上时，建议将其更新为侧边的导航栏，以提升人体工学体验。在超大窗口中，可以考虑展开导航栏以显示标签。

精心设计的响应式布局能确保你的应用在桌面体验中看起来很棒，但要让应用真正感觉"桌面级"，你还需要针对鼠标、触控板和实体键盘等桌面输入方式进行设计。这意味着不仅要兼容，还要充分利用它们独特的特性。

对于鼠标和触控板而言，兼容意味着所有可以点击的元素都能被点击，并且显示合适的悬停状态来指示可点击区域。但还有更多方法可以进一步打磨体验，下面是其中几个：

- 你可以显示不同的光标图标，更清晰地指示底层元素的功能，比如 用中文口语翻译如下：

用 I 型光标来显示可选文本。用指针设备的次级点击（鼠标右键点击，或者触控板双指轻点）来调出那些通常在触屏上靠长按或者滑动手势才能触发的上下文选项。用光标悬停事件，让用户方便地访问一些补充操作，又不会把界面塞得太满。

关于实体键盘支持，要做到完全兼容，你应用里的每一个可交互组件都必须能用 Tab 键或方向键访问到。到了之后，用户得能用回车键或者空格键来触发这些元素，而且要有一个清晰、明显的焦点样式，让人一眼就能看出当前选中了哪个。

为了让键盘更好地提升效率，还要支持一些标准的快捷键，比如 Ctrl+C 和 Ctrl+V 这种复制粘贴的，或者你应用里任何能帮用户更快完成事情的快捷键。屏幕上那个链接是键盘相关的文档。

刚才快速过了一遍从一开始就把桌面体验考虑进去的设计流程。接下来我把讲台交给 Francesco，让他跟大家详细聊聊能帮你们实现这些设计的 API 和开发者工具。

>> 谢谢 Jason。当用户在桌面类的环境里打开你的应用，他们期待的是和其他地方一样的能力。你的应用是创造力和生产力的入口，用户想专注于手头的事情，而不是操心用的什么平台。我们来看看 Android 提供哪些现代工具和 API 来打造超棒的桌面体验。

Jetpack Compose 特别擅长做自适应应用，因为它声明式、状态驱动的架构能动态地重组 UI，顺滑地适配任何窗口大小。配合窗口尺寸类别和导航三件套，开发者能轻松搭出在不同形态的设备上都表现出色的布局。这点很关键，因为做桌面端设计时，应用的观感会从根本上不一样。真正的桌面应用从一开始就把窗口当一块大画布。功能不再藏在层层导航后面，而是优先考虑高信息密度——用常驻的工具栏、悬浮面板、可停靠的侧边栏和上下文菜单，把工作站的全部实力毫无保留地摆出来。

不过，光做出好看的 UI 只是第一步。要真正做出桌面级的体验，你的应用必须在桌面生态里表现得像个一等公民。咱们先从输入说起。Jetpack Compose 用自带的声明式修饰符和焦点管理，把硬件事件和传统的触控一起处理，大大简化了输入支持。Compose 1.11 大幅改进了对触控板的支持：触控板事件现在普遍会被识别成 pointer type.mouse（鼠标类型）而不是 pointer type.touch（触控类型），让鼠标和触控板的行为更符合用户预期；还加入了对更复杂的触控板手势的支持，比如双指滑动和捏合；另外也引入了新的测试 API，比如 performTrackpadInput，方便你验证应用在用触控板操作时的行为。

另一个重要方面是，桌面的工作流程天然是相互连通的，用户不是孤立工作的。举个例子，你应该实现对文字和图片的富拖拽支持，打破应用和外界之间的那堵墙，让它能参与到更广泛的桌面工作流里去。

接下来想想用户怎么操作。在手机上，一次做一件事是常态。但在桌面类的环境里，用户可能想同时打开两份不同的文档并排放着对比。如果你的应用只支持单实例，那你就是在拖他们效率的后腿。这里的架构策略是：把应用配置成支持多实例。设计好你的状态管理，让同一个应用的两个完全独立的窗口能同时运行，从同一个数据源拉数据，还不会互相覆盖状态。这个思维上的转变会彻底改变你默认的功能集。

拿编辑器应用举例。在手机上，应用通常以“阅读优先”为主，格式工具都藏起来，好把有限的屏幕空间留给阅读。想编辑的话，你得专门点一个编辑按钮切换模式。但在桌面上，假设是“创作优先”：你一打开文档，光标就已经在编辑器里闪了，所有工具都摆出来，点一下就能用。默认状态就是为“上手即用、零摩擦地干活”优化的。

这个桌面愿景的美妙之处在于，它和 Android 的技术栈是无缝集成的。就算用户在手机上先开了个头，Android 也能把那个上下文带到桌面类的环境里，让他们用更趁手的强大工具把活儿干完。

我们正在进入一个移动和桌面边界逐渐消失的时代。通过采用自适应布局、尊重桌面端的输入范式、支持多窗口功能，你做的不仅仅是一个 Android 应用，而是面向多种设备的生产力工具。去打造让人惊艳的体验吧，谢谢大家，祝你们 I/O 大会剩下的时间玩得开心！
