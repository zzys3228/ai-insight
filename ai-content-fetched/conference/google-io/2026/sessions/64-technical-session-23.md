# 在Chrome开发者工具中用Gemini突破界限

## 📋 摘要

## 📝 完整文字稿

[音乐] >> 大家好，我是 Matias，Chrome 工具团队的开发者关系工程师。在这节课中，我会带大家看看 Chrome DevTools 在 AI 创新以及其他方面的最新更新。我们将一起看看：现在的 DevTools 配合 AI 助手如何支持更自然的对话；全新的用户体验和界面如何通过结构化的引导流程，帮助你追踪 Gemini 在 DevTools 中的操作和推理；DevTools 现在如何借助 AI 和新能力帮助你在更多 Web 开发领域进行调试；以及它如何用更简洁的答案和洞察帮你更快完成工作，这些内容你还可以直接交给你的编程 Agent 使用。最后，我们还会看看 DevTools 一直以来的强项——提供直观的工具来调试 Web 的最新特性。话不多说，我们开始吧。

在聊新功能之前，先帮大家拉齐一下背景。过去几年，我们把 Gemini 的能力带到了 DevTools 中很多常见的调试流程中。我们把这些功能统称为 AI 创新，一共有这些：浏览器控制台中的"控制台洞察"，用来解释控制台里的警告和错误；"AI 助手"，让你可以直接在 DevTools 里和 Gemini 对话；控制台和源代码面板里由 Gemini 提供的代码补全和代码生成；以及"自动标注"，通过对追踪事件添加建议标签，帮助你理解复杂的性能追踪结果。你可以在 DevTools 设置中的"AI 创新"里找到并启用这些功能。如果只能选一个功能来用，那一定是 AI 助手，因为我们给它做了一次大改版。来看看具体更新。

在第一版的 AI 工具里，对话是一来一回的简单形式，就像下棋一样：你写一个提示词，然后轮到 Gemini 给答案。根据这个答案，你再写下一个提示词，这样来回进行。但这并不是开发者日常工具的工作方式。像 Gemini CLI 或 Antigravity 这种 Agent 化编程工具，只需要一个提示词，就能持续工作、写代码、调试——它们是"Agent 化"的。我们现在把这种 Agent 化对话的能力带到了 Chrome DevTools 的 AI 助手中。来看个例子。

你可以通过 DevTools 右上角的快捷方式打开 AI 助手。它会在抽屉面板里打开 AI 助手窗口。通过一个全新的上下文选择 Agent，你可以立刻基于整个页面的上下文提出初始问题。根据你的原始提示词，更专业的 Agent 会接手上场，在 DevTools 中找到相关的上下文。举个例子，你不用多想，直接问："我该如何提升这个页面的性能？"要合理回答这个问题，不管人还是 AI 都需要性能追踪作为上下文。所以，AI 助手里的 Gemini 会直接帮你重新加载页面并录制一次追踪。录完之后，它会立即开始分析这个追踪，重点关注 Core Web Vitals——也就是 LCP、INP 和 CLS，以及相关的性能洞察。Gemini 分析完所有性能数据后，会给出可操作的建议，帮助你优化页面中影响最大的 Core Web Vitals 指标。在这个例子里，LCP 图片加载超过了 2 秒，加上其他的图片资源，我总共浪费了将近 2MB——看来我确实有不少活要干。

如果你以前用过 AI 助手，你可能会注意到这个例子的另一个不同。我们把 Gemini 的回复大幅精简，让你能更快采取行动，不用再对着大段文字读个不停。平均下来，Gemini 回答你问题所生成的字数减少了 68%。这些改进能帮你用一个问题就诊断出原本需要多轮对话和手动操作才能解决的复杂问题，让调试更快、更有信心。

Gemini 帮你做更多事固然很好，但和开发者们交流之后我们了解到，大家还是希望自己能掌控全局，搞清楚到底发生了什么。所以我们花了额外的精力，确保你能通过结构化的引导流程，跟着 Gemini 一起理解它对你网站的推理过程。要看看它怎么运作，我们回到刚才的性能例子。

Gemini 录制了追踪文件并总结了它的发现。现在，全新的"Agent 引导流程"可以让我们一起看看背后的过程。点击"显示 Agent 引导流程"，一个分步骤的引导流程会在 AI 助手面板的分屏视图中打开。流程里的关键步骤会通过专门的控件展示，一眼就能看懂。比如，作为 Gemini 性能调查的一部分，它查看了页面加载过程中记录的 Core Web Vitals。点击控件里的"查看"按钮，你会立刻跳转到 DevTools 中信息来源对应的位置。这让你能轻松跟上 Gemini 在 DevTools 中的每一步，自己也能跟着重复这些步骤。

目前，我们为 DevTools 中各种调试场景提供了六种不同的控件。它们可以展示计算样式、Core Web Vitals……（未完） 好的，我来用口语化的中文翻译这段内容：

……一些细节信息，比如简化的 DOM 树、LCP（最大内容绘制）拆解分析、第一方和第三方的性能总结等等。而且只要你点一下"查看"，就能直接跳转到 DevTools 里原始的数据来源，把 AI 智能体的分析结果和你自己手动调查无缝衔接起来，顺带还能让你熟悉一下 DevTools 的界面。

我们之前已经简单聊过对话上下文这个事了。你现在不需要先去选中某个具体的元素、网络请求或者源代码文件，才能开始跟 AI 助手对话。DevTools 会自动帮你找到合适的上下文。而且 AI 助手现在能用更多的工具和数据源了，能帮你找到更相关的上下文来回答你更多的疑问——我们一直在扩展它的能力范围，以后也会继续这么做。

举个例子，你们看到那些让人头疼的圆环分数了吗？那是 Lighthouse（灯塔工具）打出来的分。现在 AI 助手也能看到这些分数了，这一下子就能回答一大类新问题了。我们来看个例子。

现在你可以直接问 AI 助手："我该怎么提升这个页面的无障碍访问性啊？" 有了新的智能体能力，Gemini 和 AI 助手就会去调取 Lighthouse 的审计报告，就像处理性能问题一样，根据 Lighthouse 的无障碍审计给你一些能落地的改进建议。碰到像颜色对比度这种问题，你还可以让智能体在被检查的页面上直接动手修一下，然后再让它验证一下修改效果，比如跟它说："帮我修一下颜色对比度的问题，然后确认一下改完之后的情况。" 这种方式挺适合快速验证一下效果的，但问题来了——怎么把这些改动同步到你真实的代码里去呢？这就要介绍一个新功能了：**一键复制给你的编程 AI 智能体**。

回到刚才那个例子。在每次对话结束时，点一下"复制给你的编程 AI 智能体"这个按钮，Gemini 就会把整个调查的结果和可能的修复方案总结一下，告诉你这些数据是从哪儿来的，然后把所有内容压缩成一个可以直接用的提示词。你可以把这个提示词复制下来，从 DevTools 里拿出来，粘贴到你常用的编程 AI 智能体里——比如这个例子里演示的 Gravity，或者别的什么工具都行——然后让它帮你把活儿干完。

我们在扩展 Chrome DevTools 里 AI 助手的能力上真的下了很大功夫。当然，有些事情它确实帮不上忙，比如帮你决定晚上吃什么 😄。但如果你觉得有什么能力是缺的，欢迎在评论区告诉我们。

Chrome 工具团队一直引以为豪的一点就是：网页平台出了什么新功能，我们都能迅速给你开发出配套的工具来用。不管是视图过渡（view transitions）、新的 CSS 规则（比如 @starting-style）、CSS 轮播图、圆角形状（corner shape），还是最新的 masonry 网格规范——在这些功能还没成为基线标准、还没广泛普及之前，你就已经能用 DevTools 来调试它们了。我们以后也会继续这样做：网页平台每加一个新功能，DevTools 就第一时间支持它。

而且这个清单里现在又多了一项新功能：**WebMCP**。WebMCP 是一个正在提议中的网页标准，它让网站可以给来访的 AI 智能体提供专门的工具。这样 AI 智能体就能更顺畅、更高效地跟网站交互了。你可以在"应用"面板里找到 WebMCP 的新工具，只要在侧边栏里选中 WebMCP 就行了。从这里，你可以查看发现的工具、追踪工具的调用情况，或者手动调用工具来调试结果，不用非得通过 AI 智能体来操作。

想深入了解 WebMCP 的话，可以去听我同事 Casper 的分享——"为智能体时代构建你的网站"那个环节，他会讲得很详细。

现在来总结一下今天讲的内容。我们看了 DevTools 新的智能体能力，能让你调试得更快、更全面。带小部件的 AI 引导功能让你始终掌握主动权。AI 助手突破了以前的边界，现在能帮你调试和优化更多方面了，比如无障碍访问性。如果你更喜欢用自己的编程 AI 智能体来做改进，还可以通过"导出给智能体"功能把调试结果交给它。还有新的 WebMCP 调试能力，这体现了 DevTools 的一个核心理念：**平台在不断进化，我们就迅速给你提供配套的调试能力**。

所有这些功能今天就能用了。我建议你赶紧去 DevTools 设置里把 AI 创新功能打开，自己亲身体验一下它们会怎么改变你的工作流程。
