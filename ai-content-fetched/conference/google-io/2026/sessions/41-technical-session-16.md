## 会议演讲总结

**主题：使用 Jetpack Media3 和 CameraX 提升 Android 媒体体验**

本次演讲由 Google Android 媒体团队的产品经理 Bassam、开发者关系工程师 Nevin 及工程团队 Kai Yang 共同分享，介绍了 Android 在媒体体验方面的最新进展和工具更新。

**核心要点：**

1. **整体愿景**：媒体已从单纯消费内容转变为创作、连接和生活的载体，团队致力于在画质、功耗、性能、创作工具和空间媒体等高影响力领域持续投入。

2. **CameraX 新功能**：从 1.5 版本起支持原始 DNG 图像捕捉和高帧率视频拍摄；新增功能组（Feature Group）API，可智能识别设备能力并启用最佳组合；一行代码即可启用硬件加速的暗光增强功能（Instagram 已采用）。

3. **自适应拍摄体验**：CameraX 取景器（Viewfinder）Composable 组件可自动适配折叠屏、平板等多种设备形态；结合 Jetpack 画中画库，支持录制时多任务操作。

4. **未来方向**：CameraX 将支持动态照片拍摄，Media3 已支持动态照片的编辑和播放。

5. **Media3 Transformer**：支持多素材视频编辑，提供特效、合成预览和导出等完整工具链，将复杂编辑流程化繁为简。

[音乐] 大家好，欢迎来到我们的技术分享会，主题是"为你的 Android 媒体体验注入超强动力"。我是 Bassam，Android 相机和媒体方面的产品经理。今天跟我一起的还有来自开发者关系的 Nevin，以及我们工程团队的 Kai Yang。我们非常激动地向大家展示 Android 是如何不断进化，帮助你为用户打造更具吸引力的体验。让我们开始吧。

我们与媒体的关系已经发生了根本性的变化。媒体不再只是我们消费的东西，它更是我们创造、连接和生活的方式。越来越多的，我们看到媒体已经成为 Android 中至关重要的元素。随着这种关系的演变，我们也看到用户在使用设备观看、聆听、创作、分享和连接时，深度地沉浸在自己的媒体中。

这就是为什么我们希望你的媒体体验能够毫不费力。你观看的每一个视频、听到的每一首歌曲、喜爱的每一个故事，都应该以出色的、不间断的画质呈现出来。与此同时，我们也承诺成为你的创作伙伴，让你能轻松地记录生活中的回忆，并打造只有你才能讲述的故事。我们致力于呈现不只是让你看到，而是能让你置身其中的媒体。

为了实现这一愿景，我们正在围绕几个高影响力领域进行重点投资。首先，我们持续大力投入画质、功耗和性能，确保 ExoPlayer 能帮助你在 Android 上提供最省电、稳定可靠、毫无卡顿的媒体体验。

我们还专注于端到端的创作之旅，让应用能够利用我们领先的智能相机，配合无缝衔接的 CameraX 工具包和 Media3 编辑库，为最有创意的用户解锁神奇体验。

最后，我们正在投资打造用户能"感受得到"的沉浸式体验。当内容突破平面屏幕的束缚时，我们正在构建空间媒体的终极平台。通过我们开源的 Eclipse 引擎，我们能够让你在 Android 设备上捕捉和播放空间内容。

在核心投资的基础上，我们也为生态带来了令人自豪的创新功能。与 Meta 等合作伙伴紧密协作，我们成功地将"暗光增强"等功能带给了众多用户。我们还为平台带来了更多创新性的 AI 功能，比如魔法橡皮擦和录音棚音效，让最有创意的用户拥有自由表达的工具。

除了我们正在启用的所有这些体验之外，我们还非常自豪地看到 Jetpack 库持续被广泛采用。在整个技术栈中，我们会分享更多关于这些库如何帮助你利用 Android 最新相机和媒体功能的内容。

要深入了解细节，我把话筒交给 Kai Yang 和 Nevin。

谢谢 Bassam。当我们谈论"在任何场景下都能捕捉"时，我们的意思是让你的用户每次都能毫无障碍地拍出完美的照片。我们致力于通过 CameraX 弥合系统能力与应用内功能之间的差距。CameraX 抽象了底层相机硬件的复杂性，提供了一致且强大的 API 界面。

我们来看看有什么新功能。例如，从 CameraX 1.5 版本开始，你可以捕捉未压缩、未处理的 DNG 图像，为专业级编辑提供最大的创作灵活性。

在视频方面，我们引入了一个备受期待的功能——高帧率拍摄。你可以以高帧率捕捉快速运动的画面，并按原帧率编码，实现极其流畅的视频；或者直接以较低的帧率进行编码，生成富有戏剧效果的慢动作视频。

关于代码示例、设备兼容性和限制的详细信息，请查看我们关于这个功能的博客文章。

现代相机体验通常会组合多个功能。但是，你怎么知道某个设备是否同时支持 HDR 视频、60 帧每秒以及预览防抖呢？CameraX 中全新的功能组（Feature Group）API 解决了这个难题。功能组 API 目前支持 HLG 10 HDR 视频、60 帧每秒、预览防抖以及 Ultra HDR 图像。你可以提供一个按优先级排序的功能列表，CameraX 会确定设备支持的最佳组合，并按你列表的顺序启用它们。你可以考虑使用这个 API 来创建设置界面，自动禁用设备不支持的选项的开关。

更多使用指南，请访问我们的功能组博客文章。

接下来，让我们来看看暗光增强（Low Light Boost），这是一个非常强大的功能，可以在预览和录制时实时提亮相机画面。Instagram 最近增加了对暗光增强的支持，帮助用户在夜间以尽可能真实的细节捕捉回忆。使用 CameraX，只需一行代码，在你的拍摄流程中调用 `cameraControl.enableLowLightBoostAsync` API，就能实现硬件加速的暗光增强 AE 模式。

请查看我们的开发者文档，了解如何在不支持暗光增强 AE 模式的设备上启用软件实现作为后备方案。

到目前为止，我们介绍了 CameraX 如何让你的拍摄体验适应不同环境，从快速运动到暗光场景。但现在用户的设备种类繁多，包括折叠屏、平板、头显和眼镜。因此，构建一个能无缝适应新形态的相机体验变得前所未有的重要。下面由 Kylie Ying 来为大家演示。

最后再说说未来的方向，CameraX 在新格式支持方面一直走在前面，这次又加了动态照片功能。今年晚些时候，你就能用 CameraX 在你的 App 里实现动态照片的拍摄了，而 Jetpack Media3 已经支持动态照片的编辑和播放了。说到编辑，接下来我们展示一下 Jetpack Media3 里的 Transformer API 是怎么把那些有创意又复杂的编辑变得简单的。Transformer 的一个重点就是支持更复杂的编辑流程，现在的 API 已经支持多素材编辑了。多素材编辑就是把好几个输入的媒体素材合成成一个输出视频的一整套工具。比如，可以把素材一个接一个拼起来，或者叠在一起，或者安排它们同时播放，还能对跨素材的操作进行处理。

在 Media3 里，你可以用特效 API 给媒体加特效和变换，用合成播放器预览完整的合成效果，最后用 Transformer 导出成视频文件。要给单个素材加特效，可以创建一个编辑过的媒体项，它把一个媒体和一堆要应用的视频特效、音频处理器打包在一起。你可以把多个编辑过的媒体项串成一个编辑序列，再把多个序列叠在一起变成一个合成。合成层面还能加一些编辑选项，比如 HDR 行为之类的。然后你可以用合成播放器来预览编辑效果，或者把它传给 Transformer 导出成带编辑效果的成品视频。

除此之外，Transformer 还塞了一堆新功能和改进：比如用编辑列表元数据来加快裁剪速度，你只要标好起点和终点用元数据就行，不用重新编码整个片段就能快速生成高质量的短视频片段；新增了像 Android 专业视频这样的输入格式支持，给专业级编辑提供更多控制；默认用 Media3 的封装器而不是平台自带的，这样在不同设备上表现更一致；还有更直观的 API 来调整速度和帧率，这些都在编辑媒体项的构建器里。

有个用上了 Transformer 的开发者就是 Adobe。他们即将推出的 Adobe Premiere App 就是用 Transformer 作为媒体编辑基础设施的底子，这样能降低维护成本，在整个 Android 生态里获得更好的兼容性，同时还能保留他们自己定制图形管道的灵活性。
