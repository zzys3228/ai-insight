## Web UI 新特性总结

本次 Google I/O 2026"Web UI 新特性"演讲由 Chrome DevRel 团队成员 Yuna 和 Ramus 共同分享，重点介绍了提升网页用户体验的最新 Web 能力。

**核心要点：**

1. **五大用户体验原则**：作为现代 Web 体验标杆，包括尊重用户偏好、实现自然交互、提供引导式导航、最大化内容展示、减少视觉干扰、适应不同设备形态。

2. **尊重用户偏好的新 API**：
   - **`contrast-color` 函数**：根据 WCAG2 对比度算法，自动返回与背景色对比度更高的黑/白色，已纳入 Interop 2026，达到新增可用基线。
   - **`light-dark` 函数**：在一条声明中同时指定浅色和深色值，最新规范更新后还支持图像参数（Chrome 150）。
   - **样式查询（Style Queries）**：根据父元素的样式值（如主题）为子元素添加样式。
   - **`if` 函数**：允许在属性值内直接编写条件逻辑。

3. **全新开发范式**：**Canvas 中的 HTML**，将可访问的 DOM 内容带入 Canvas 体验，架起高性能图形与语义化 Web 之间的桥梁。

演讲强调，这些能力让 Web 体验更"高品质"，同时降低开发者实现动态个性化的门槛。

其他的 CSS 能力。CSS 自定义函数和样式查询。结合起来，你可以创建支持任何值类型的自定义 light-dark 函数。要实现这个，首先你需要把颜色主题的值存到一个自定义属性里，比如 d-scheme，就像左边这里展示的。默认情况下，它被设置成 light，但当操作系统切换到深色模式时，这个值就会变成 dark。然后在右边这个自定义函数里面，你可以根据条件让结果要么用 light 参数，要么用 dark 参数。默认情况下用的是 light 的值。但同样，当 d-scheme 现在被设置成 dark 的时候，就会用 dark 的值。这个就是用样式查询来实现的。最后，就像你调用原生的 light-dark 函数一样，你现在可以调用你自己定义的这个 light-dark 函数了，而且用值的类型没有任何限制。

跟所有容器查询一样，你只能根据父元素的值来给子元素加样式。但如果你想给元素本身加样式，你可以在里面再加入第三个新的语言能力，那就是 if 函数。这个函数让你可以直接在属性值里面写条件逻辑。如果报告的 theme 是 dark，结果就设置为 dark 参数的值，否则就设置为 light 参数的值。你可以通过访问链接，或者收听 CSS Podcast 的第 92 集，了解更多关于 if 函数的详细信息。如果你还没听过 CSS Podcast，强烈推荐你去听听，你可能会听到一些熟悉的声音。

回到刚才那个 demo，我们来把这些现代的 CSS 能力——contrast-color、style queries、light-dark 全部组合起来，把我们的 elevation 样式和配色方案都自定义一下。你可以看到，这里的 elevation 机制也用了一个函数，根据颜色主题在 shadow 和 neon glow 之间切换，用的就是 light-dark 函数。主题颜色是用 style queries 设置的，然后用 contrast-color 函数来确定最终选中的颜色。这个函数的结果要么是黑色，要么是白色。所以当你把它存到自定义属性里的时候，你就可以把它当成布尔值来用，用这个 if 函数配合 style queries，把结果映射到任何值。CSS 真的越来越强大了。Contrast-color 让你可以把页面的颜色主题和卡片本地的主题分开，而 style queries 给了你在输出值上的灵活性。所以你可以让卡片在深色模式或者浅色模式下都有一个更亮的背景。但前提是你测试过所有颜色的组合，这样你就能始终保证有合适的颜色对比度，让这个主题非常动态。

为了让你对自定义有更多的控制权，CSS 工作组最近决定给 @container、@property、@function 这些 @ 规则加一个 @supports 查询。你可以在 @supports 后面用 @supports 函数，把要检查浏览器支持的 @ 规则字符串放进去，比如 @function。最后一个细节，当你访问这个 demo 的时候你可能根本不会注意到：这个页面会尊重用户在操作系统辅助功能里设置的字体大小偏好。这只是一个很小的调整，但影响很大，因为最近的数据显示，在 Android 和 iOS 上，超过 30% 的用户都修改过系统设置里的默认字体大小。

要给网站加上这个功能，有一个新的 meta 标签叫 text-scale，Chrome 146 已经支持了。如果你把值设置为 scale，就等于告诉浏览器你的网站可以处理这些放大的字体。设置之后，HTML 元素的字体大小就由操作系统来决定，所以你就不需要自己设置基础字体大小了。然后如果你用相对单位的长度，比如 em 和 rem，那些计算出来的像素值就是基于这个基础字体大小来的。

要在你开发网站的时候测试这个，DevTools 里有一个选项可以模拟。你可以在 Rendering 标签页里找到它，然后通过下拉菜单来改变值。
