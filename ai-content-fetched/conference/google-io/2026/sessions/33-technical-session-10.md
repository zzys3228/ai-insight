## 📝 内容总结

**会议演讲总结：在 Android 上构建一个现代的笔记应用**

本演讲由 Android 开发者关系团队的 Joyce 和 Chris 共同主讲，重点介绍了如何利用 Jetpack Ink 库为 Android 设备（尤其是支持手写笔的大屏设备）打造流畅、现代的笔记体验。

**核心要点：**

1. **Jetpack Ink 库的六大模块**：(1) 笔迹模块处理几何计算；(2) 排序模块基于低延迟库实现接近零延迟的实时输入捕捉；(3) 渲染模块通过优化图形管线高效绘制；(4) 几何模块提供选区、自定义橡皮擦等数学工具；(5) 笔刷模块定义视觉样式；(6) 存储模块采用 protobuf 和差分压缩算法高效序列化数据。

2. **应用集成示例 Cay**：作为完整的开源笔记应用，Cay 演示了 Ink 的核心用法——用 Compose 组件实现低延迟绘制，用 progress strokes 功能处理"湿笔迹"实时渲染，通过 onStrokeFinish 回调将"干笔迹"存入 ViewModel，再由 CanvasStrokeRenderer 在画布和静态预览中高效显示。

3. **生态集成**：除绘制外，应用还需结合桌面级窗口、多任务、系统级笔记入口和拖放等系统功能，才能全面适配现代 Android 设备形态。

该方案对所有希望提升创意与生产力体验的应用都具参考价值。

嗨，我是 Joyce，来自 Android 开发者关系团队。今天我们来探索一下 Jetpack Ink 库的基础知识，以及各种 Android 功能，看看它们如何帮助开发者为用户打造独特而有趣的笔记体验。就算你的应用不是专门做笔记的，也别急着划走哦～ 我们今天讲到的很多关于手写涂鸦和 Android 生产力方面的内容，其实对所有想要添加创意体验、优化适配现代 Android 生态系统的应用都很有用。Android 已经从一个只能在手机上用的操作系统，发展成了一个支持各种设备的庞大生态。今天，我们要一起庆祝这种设备的多样性，也希望开发者们能跟上这个节奏。

特别是那些支持手写笔等额外输入方式的大屏设备，简直就是为笔记、办公这类生产力场景量身定做的。手写笔的输入能把这些大屏设备变成电子笔记本，让用户只用一台设备就能搞定所有笔记、待办事项和涂涂画画。

当然，一个好用的笔记应用，最核心的就是那种自然、流畅的书写手感。Jetpack Ink API 可以让你轻松画出漂亮的手写笔迹，而且我们的"接近零延迟"技术，真的会让你感觉笔迹就是从手写笔上自然流出来的。更有意思的是，你还可以自定义笔刷，给用户带来无穷的乐趣和创意。

除了书写体验本身，还有一些很关键的生产力功能和系统集成，能让应用显得更现代、更独特，能完美适配 Android 生态里的各种设备和形态。比如桌面级窗口、多任务、多开这些功能，能让用户效率翻倍。还有系统级的笔记角色入口、拖放交互这些，能让数据转移和应用启动变得更顺滑。

接下来我们就动手实操，一起来把这些想法落地。我们会逐个深入讲解这些概念，并展示 Chris 帮忙搭建的真实的笔记应用中的实际例子。

谢谢 Joyce。大家好，我是 Chris，Android 开发者关系工程师。想要做出流畅、自然的书写体验，以前是真的难。想要那种"笔在纸上画"的感觉，你得自己搞定复杂的图形管线、处理贝塞尔曲线、还得自己写几何工具，就为了能擦掉一条线。所以我们才开发了 Jetpack Ink API。

我们的目标很简单：把那些复杂的图形和输入处理都包圆了，让你们能专注做自己独特的酷功能。我们把它设计得非常模块化，你可以整套都用，也可以只挑你需要的某一块用。举个例子，Android 自己的"圈选搜索"功能，就用了 Ink 的创作和笔刷模块，精准地捕捉输入并画出那道标志性的发光圆圈。

Ink API 分为六个模块。第一个是 **strokes（笔迹）模块**，这是基础，负责处理输入批次和所有跟手写相关的几何计算。第二个是 **ordering（输入排序）模块**，它负责实时捕捉指针输入——这里大概率就是手写笔了——而且能做到接近零延迟，因为 Ink API 是基于我们之前发布的低延迟库和运动预测 API 搭起来的。然后是 **rendering（渲染）模块**，它通过优化的图形管线，把你的笔迹高效地画到 Android 画布上。**geometry（几何）模块**给你提供了数学基础工具，方便你做选区工具、自定义橡皮擦之类的东西。**brush（笔刷）模块**则定义笔迹的视觉样式、粗细和颜色。最后是**最新的 storage（存储）模块**，它用 protobuf 和差分压缩算法来高效地序列化笔迹数据。

来看看我们的合作伙伴怎么用 Ink API 的吧。Google Docs 用它做了新的智能标注功能；RV App Studio 的 PDF 扫描器也用了它，实现了低延迟的文档批注。

你可能会问：到底怎么把它用在项目里呢？来认识一下 **Cay**——我们做的一个完整的、离线优先的笔记应用示例，现在已经在 GitHub 上开源了。

在这节课里，我们会演示 Cay 怎么用 Ink API 的 Compose 组件来实现低延迟绘制，怎么实现丰富有趣的自定义笔刷，以及怎么深度集成系统功能，让整体体验更上一个档次。

我们先来看绘制画布。在 Cay 的绘制界面里，我们用了 Ink 的 Compose 组件和创作模块里的 progress strokes（进行中的笔迹）功能，来处理实时的手写笔和触摸输入。这个模块会捕捉指针事件，并以尽可能低的延迟渲染"湿笔迹"——也就是用户当前正在画的那一笔。你只需要把当前的笔刷状态传给它，再加一个画完之后的回调就行了。

用户一抬笔，这一笔就算画完了，也就是"干"了。这时候 onStrokeFinish 回调就会触发，strokes 模块会给你一个最终的、不可变的笔迹对象，应用就能把它存到 ViewModel 里去了。

要显示已经画好的笔迹和新画完的笔迹，Cay 用的是渲染模块里的 CanvasStrokeRenderer，它能把笔迹对象高效地画到标准的 Compose 画布上。这个组件我们在两个地方用：一个是在绘制界面里实时画画的时候，另一个是在首页的笔记详情里给笔迹做一个静态预览。

那如果用户画错了，想擦掉一笔呢？这时候就该 **geometry（几何）模块**出场了。在 Cay 里，当用户选中橡皮擦工具时，系统会在用户手势的路径周围创建一个可变的平行四边形。橡皮擦会检查这个形状和现有笔迹的边界框有没有相交，从而精确地判断要擦掉哪些部分。整个操作既直觉又精准。

最后，我们需要把这些画的内容保存下来，这样下次打开应用时还能看到。打个比方，单单一笔可能就包含几百个数据点。你当然可以简单粗暴地直接存，但这样一来你的数据库会膨胀得非常快，有时候甚至会直接崩掉……
