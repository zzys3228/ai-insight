## 📝 内容总结

**《在 Android Studio 中借助 AI 更快地开发》核心要点**

一、核心主题概述

本演讲全面介绍Gemini在Android Studio中的代理能力，涵盖三大核心支柱：原型与功能构建、测试与维护、工作流集成和Android Skills。

**二、关键发布/技术要点**
- 要点1：新项目Agent——一句话提示即可搭建完整Android应用，遵循最佳实践
- 要点2：规划模式（Planning Mode）——复杂任务先制定详细实施计划，用户可反复调整
- 要点3：多任务并行——支持同时运行多个对话，等待长任务时可快速提问
- 要点4：MCP集成——连接GitHub等外部系统，智能体可自主处理issue
- 要点5：Journey测试——动态适应UI变化的端到端测试，支持智能体修复失败
- 要点6：Android Skills——模块化指令包，让AI获得最新API和最佳实践知识
- 要点7：AGP升级助手——自动化迁移到新版本AGP DSL

**三、落地应用**
- 案例1：健身追踪App从一句话提示到完整可运行应用
- 案例2：Agent模式10分钟内完成项目搭建
- 案例3：XML迁移到Compose的Skill帮助开发者逐步转向Compose优先

**四、总结**
Android Studio的AI代理能力覆盖了从项目创建到发布维护的全生命周期。通过Skills、MCP和Agent模式，开发者可大幅提升效率，同时保持对最新Android框架和最佳实践的应用。

Skills 就是一个开放标准的 Markdown 文件（skills.md），你可以把它理解为给 AI agent 和工具用的模块化指令集。它给某个任务提供了技术规范，把最新的、领域内的信息喂给大模型，让模型有据可依。它让 AI 能通过额外的知识和上下文，去执行那些需要专业能力的工作流。

那平时怎么用呢？一般来说，你从一个靠谱的地方把 skills 拉下来，放到一个专门的目录里，比如本地项目根目录或者用户主目录。以 Android Studio 为例，skills 可以放到工作区专属的目录里，也可以放到全局目录里。

放好之后，agent 就会知道有哪些 skills 可用——它会维护一个 skills 的元数据列表，每次你发 prompt 的时候都把这些元数据发给大模型。当你（也就是用户）发了 prompt 之后，大模型就会拿 prompt 的内容去跟这些 skills 的元数据和描述做匹配，找到最合适的那一个然后把它激活。

Skills 设计成是自动触发的，所以你不用自己去手动搜、手动挂载、非得让 AI 用它——当然你想强制用也行。

好，基础的差不多讲完了。那对于 Android 开发，咱们怎么把这个东西用好呢？你在开发者效率主题演讲里应该听过这个了，但为了更好地支持大家用 AI 做 Android 开发，我们推出了官方的 Android skills——一个专门给 AI 优化的指令仓库，帮模型更好地理解和执行那些符合我们 Android 开发最佳实践的特定模式。

要方便地装上这些 skills 并且保持更新，我们推荐用 Android CLI，具体可以去看它的文档。当然你也可以去 GitHub 仓库看最新的动态。

搞 Android skills，其实是想解决我们发现的大模型在实际使用中的几个痛点：

首先，大模型经常因为训练数据的截止日期，对最新的 Android 框架和变更一无所知。你让它去迁移一个老旧的导航代码，它根本不知道最新的框架是啥样的。有了 skills，我们就能把这些最新的改动和更新立刻送到 AI 工具手里。

其次，你让大模型去分析一个项目的设置，它给的回答往往是零零散散的、不成体系，不一定真的帮上啥忙。Android skills 能引导大模型去执行那些更复杂、但又特别重要的工作流，同时还能遵循我们的最佳实践。

传统的文档是描述性的、概念性的、偏高层面的——拿来学习特别合适，但拿给大模型用就不太行了，因为大模型需要的是精确的、能直接照着干的具体指令。标准文档是教人怎么想、为啥这么想；而 skills 是告诉 AI 怎么去做、做啥、怎么做。

知道了这些之后，我们重点是去发布那些我们测试下来发现大模型特别需要"接地气"和"被引导"的场景的 Android skills。比如 AGP 9、XML 迁移到 Compose、给项目加 Navigation 3、从旧版 Nav 迁移、分析你的 R8 配置、开启全面屏（edge-to-edge）支持等等。

Android skills 大致可以分成这么几类：

一类是流程型的——就是一步步教你怎么完成一个特定任务的自定义流程，比如 R8 配置分析那个 skill 就是这种。

还有一类是参考和知识型的——就是一些资源合集或者指向其他支持文件的指针，给大模型补充更多知识和专业能力，比如那个 edge-to-edge 的 skill 就是教模型怎么去实现全面屏支持的。

还有一种是混合型的——把流程型和知识型整合在一起，比如 XML 迁移到 Compose 的 skill 就是这种。

Android skills 在 Android Studio 和 Android CLI 里都支持，而且它也设计成可以在任何支持 skills 的 AI 工具或者 agent 里复用。所以不管是 Google Anti-gravity、AI Studio 还是别的啥工具，都可以用。

对于 Android Studio，有一部分精选的 Android skills 是默认预装好的，你只要在设置里启用一下就行。这些 skills 在 agent 启动的时候就有了，会自动激活，所以你不用自己去手动下载或者激活。当然，你也可以自己去我们仓库或者通过 Android CLI 拉更多的 skills 下来，放到 Studio 指定的目录里。具体可以看文档了解详情。

Android CLI 也完全支持 Android skills。你可以把 skills 装到指定 agent 的专用目录里，也可以指定要用哪些 skills……
