## 简介[​](#introduction "链接到简介")

AITP 使 AI 智能体能够在信任边界之间安全地进行通信，同时提供可扩展的机制以支持结构化交互。

**设想这样一个场景：** 你让 AI 助手帮你预订一张飞往迈阿密的机票。你的助手无需逐一浏览航空公司网站，而是可以直接通过标准化协议与航空公司的预订智能体进行通信。你的助手与航空公司智能体之间交换有关航班选项、乘客信息和支付信息的结构化数据——无论你选择哪家航空公司，都使用相同的协议。

**AITP 的核心优势：**

- **通用通信**：任何兼容 AITP 的智能体都可以相互交互，无论由谁构建
- **结构化数据交换**：除文本外，智能体之间还可以共享 UI 元素、表单和支付请求
- **跨越信任边界**：实现代表不同实体的智能体之间的安全交互
- **可扩展能力**：可通过能力系统添加新的交互类型

我们设想，在未来，大多数在线交互都将由代表个人、企业和政府机构的 AI 智能体执行，它们与用户及彼此之间进行通信。这些智能体将结合当前在线服务的规模效益和成本优势，同时具备人类交互的灵活性和个性化体验。正如 HTTP 和 HTML 使任何网页浏览器都能访问任何网站一样，AITP 为智能体对智能体以及用户对智能体的通信提供了统一标准，无论这些智能体运行在何处或如何构建。

要深入了解 AITP 旨在解决的问题以及我们对智能体交互未来的愿景，请参阅[愿景](/vision)页面。

## 协议概述[​](#protocol-overview "链接到协议概述")

AITP 由两部分组成：

1. 一个核心协议，用于在 [**聊天线程**](/threads) 中与智能体进行通信，灵感来源于 OpenAI Assistant/Threads API 并与之基本兼容。了解更多：
   1. [为何使用线程？](/threads#why-chat-threads)
   2. [线程传输协议](/threads#thread-transports)
   3. [线程规范](/threads#thread-specification)
2. 一组在聊天线程上通信的[**可扩展能力**](/capabilities)，用于表明智能体的客户端（即用户界面或其他智能体）可以支持有用的标准化功能，例如多模态输入、生成式 UI、支付和/或人工介入认证。了解更多：
