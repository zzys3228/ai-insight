# OpenTelemetry 遥测开放标准

OpenTelemetry 是一个面向云原生软件的开源可观测性框架。它提供了一套统一的 API、库、代理和采集器服务，用于从您的应用程序中获取分布式追踪和指标数据。

OpenTelemetry 吸取了 OpenTracing 和 OpenCensus 项目多年的经验，并结合了社区中最优秀的理念与实践。

## 厂商无关的插桩方式

使用 OpenTelemetry API 和 SDK 对代码进行一次插桩，即可将遥测数据导出到任何可观测性后端——Jaeger、Prometheus、商业供应商或您自己的解决方案。切换后端时无需修改应用程序代码。

## 统一的可观测性信号

通过贯穿整个请求路径的共享上下文，将追踪、指标和日志关联起来。全面了解应用程序在所有组件和服务中的行为。

## 任意环境运行

OpenTelemetry 100% 开源且厂商无关。可在本地环境、混合环境或多云环境中部署，拥有完全的灵活性且无锁定。可将工作负载迁移到任何需要的地方。

## 可观测性信号

OpenTelemetry 功能特性包括：

- 自动插桩（Auto-instrumentation）
- 采集器管道（Collector pipeline）
- 上下文传播（Context propagation）
- 多语言支持（Multi-language support）
- 稳定且可用于生产环境（Stable and production-ready）
- 开放规范（Open specifications）
