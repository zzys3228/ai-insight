# OpenTelemetry 遥测开放标准

OpenTelemetry 是一个面向云原生软件的开源可观测性框架。它提供了一套统一的 API、库、代理和收集器服务，用于从应用程序中采集分布式追踪和指标数据。

OpenTelemetry 融合了 OpenTracing 和 OpenCensus 项目多年的实践经验，并结合了社区中最佳的思想与实践。

## 厂商中立的插桩

使用 OpenTelemetry API 和 SDK 只需对代码进行一次插桩，即可将遥测数据导出到任何可观测性后端——无论是 Jaeger、Prometheus、商业供应商，还是您自己的解决方案。切换后端无需修改应用程序代码。

## 统一的可观测性信号

通过贯穿整个请求路径的共享上下文，将追踪、指标和日志相互关联。全面了解应用程序在所有组件和服务中的行为。

## 任意环境运行

OpenTelemetry 100% 开源且厂商中立。可在本地环境、混合环境或多云环境中灵活部署，无任何锁定限制。可将工作负载迁移到任何您需要的地方。

## 可观测性信号

- 追踪 (Traces)
- 指标 (Metrics)
- 日志 (Logs)

## OpenTelemetry 特性

- 自动插桩 (Auto-instrumentation)
- 收集器管道 (Collector pipeline)
- 上下文传播 (Context propagation)
- 多语言支持 (Multi-language support)
- 稳定且可用于生产环境 (Stable and production-ready)
- 开放规范 (Open specifications)
