## 八、ai-news-sources.md 输出规则

> ⚠️ **重要**：输出到 ai-news-sources.md 时必须遵守以下规则

### 8.1 编号规则

```
❌ 错误：全局连续编号（会导致大量重复）
✅ 正确：分章节独立编号
```

**分章节编号示例**：
```
## 一、AI头部公司
1. NVIDIA
2. AMD
...

## 二、中立技术大会
[无编号]
...
```

### 8.2 URL类型规则

**人物URL优先级**（用于五、AI关键人物）：
| 优先级 | URL类型 | 示例 |
|--------|---------|------|
| ✅ 首选 | 个人X/Twitter | x.com/sama |
| ✅ 次选 | 个人博客/网站 | karpathy.ai |
| ✅ 可选 | 公司/机构官方X | x.com/deepseek_ai（当无个人账号时） |
| ❌ 避免 | 公司首页 | deepseek.com |
| ❌ 避免 | 院系主页 | ai.stanford.edu |

**公司/产品URL规则**：
| 优先级 | URL类型 | 示例 |
|--------|---------|------|
| ✅ 首选 | 产品页/官方文档 | platform.openai.com/docs |
| ✅ 次选 | 官方博客 | blogs.nvidia.com |
| ✅ 可选 | 年度大会 | io.google |
| ❌ 避免 | 公司首页 | nvidia.com（应指向具体产品） |

### 8.3 输出模板

```markdown
## X、分类名称

### X.1 子分类

| 源名 | URL | 推荐理由 | 状态 | 代理 | 用途 |
|------|-----|---------|------|------|------|
| 产品名 | https://产品页 | 产品说明 | ✅ | - | - |
```

### 8.4 状态标记

| 标记 | 含义 |
|------|------|
| ✅ | 可直接访问 |
| ✅302→/✅307→ | 跳转后可达 |
| ⚠️403 | 反爬限制（需代理） |
| ⚠️404 | 页面不存在 |
